# 📋 خطة تشغيل EduTrack في بيئة إلكترون وتفعيل نظام المزامنة (السودان 🇸🇩)

خطة متكاملة تهدف إلى تحويل نظام **EduTrack** إلى تطبيق سطح مكتب يعمل ببيئة **Electron**، مع تطبيق معمارية **Offline-First** باستخدام قاعدة بيانات محلية ونظام مزامنة ذكي لمواجهة مشاكل انقطاع الإنترنت والكهرباء المتكررة في السودان.

---

## 🔍 نظرة عامة وتحديات بيئة العمل في السودان
1. **انقطاع الإنترنت المتكرر**: يجب أن يعمل التطبيق بكامل وظائفه (إدخال بيانات الطلاب، تسجيل الحضور، المعاملات المالية، إدارة المكتبة) دون الحاجة للاتصال بالإنترنت.
2. **انقطاع الكهرباء**: يجب الحفاظ على سلامة البيانات محلياً وحفظ التغييرات فوراً لتجنب فقدان البيانات عند الإيقاف المفاجئ للجهاز.
3. **محدودية سرعة الاتصال**: عند توفر الإنترنت، يجب مزامنة البيانات بشكل خفيف (Incremental Sync) ومضغوط لتقليل استهلاك البيانات وتفادي الفشل بسبب ضعف الشبكة.

---

## 🏛️ المعمارية المقترحة (Architecture)

```mermaid
graph TD
    subgraph Electron Desktop Client (Local Machine)
        React[React UI Frontend] <-->|API Calls| LocalExpress[Local Express Server]
        LocalExpress <-->|Read/Write| SQLite[(SQLite Local DB)]
        LocalExpress <-->|Queue Changes| SyncQueue[Sync Queue Table]
        SyncEngine[Sync Engine Worker] <-->|Read Queue| SyncQueue
    end

    subgraph Cloud Backend (Online)
        SyncEngine <-->|Incremental Sync| CloudAPI[Cloud API Server]
        CloudAPI <-->|Sync & Merge| CloudDB[(Cloud CockroachDB)]
    end
    
    style SQLite fill:#f9f,stroke:#333,stroke-width:2px
    style CloudDB fill:#bbf,stroke:#333,stroke-width:2px
```

1. **بيئة التشغيل (Electron)**: غلاف لسطح المكتب يقوم بتشغيل واجهة React محلياً وتضمين خادم Express محلي (أو منفذ بروتوكول مباشر) للتعامل مع قاعدة البيانات المحلية.
2. **قاعدة البيانات المحلية (SQLite)**: سنستخدم قاعدة بيانات **SQLite** خفيفة مدمجة مع التطبيق لا تتطلب تثبيت أي برامج إضافية على جهاز المستخدم.
3. **طابور المزامنة (Sync Queue)**: جدول محلي يسجل كل عملية تعديل (إضافة/تعديل/حذف) مع طابع زمني وحالة العملية (`pending` أو `synced`).
4. **محرك المزامنة (Sync Engine)**: عملية خلفية تفحص جودة الاتصال بالإنترنت بشكل دوري؛ وعند توفره، تقوم بمزامنة التغييرات المحلية إلى السحابة وجلب التغييرات الجديدة من السحابة.

## 🛠️ التغييرات المقترحة بالملفات

### 1️⃣ تهيئة بيئة إلكترون (Electron Setup)
- **📄 [NEW] `electron/main.js`**: ملف التشغيل الرئيسي لـ Electron. يقوم بإنشاء النافذة وتشغيل خادم Express المحلي المدمج وإدارة دورة حياة التطبيق.
- **📄 [NEW] `electron/preload.js`**: لربط ميزات نظام التشغيل والتحقق من حالة الاتصال بالإنترنت.
- **📄 [MODIFY] `package.json`**: إضافة حزم `electron`, `electron-builder`, `sqlite3` (أو `better-sqlite3`) وإضافة سكريبتات التشغيل `npm run electron:dev` و `npm run electron:build`.

### 2️⃣ معالجة قاعدة البيانات المحلية والمزامنة
- **📄 [NEW] `server/local_db.js`**: ملف إدارة قاعدة بيانات SQLite المحلية، وإنشاء الجداول تلقائياً عند تشغيل التطبيق لأول مرة (نسخة مطابقة لجداول CockroachDB ولكن بصيغة SQLite متوافقة).
- **📄 [NEW] `server/sync_engine.js`**:
  - فحص حالة الاتصال بالشبكة.
  - إرسال طابور العمليات المحلية (Queue) إلى السيرفر السحابي لحلها وتحديث قاعدة البيانات السحابية.
  - جلب التحديثات الجديدة بناءً على طابع زمني (Timestamp) وتحديث قاعدة البيانات المحلية.
  - آلية حل التعارضات (Conflicts Resolution): الخيار الافتراضي هو **"التعديل الأحدث يفوز (Last Write Wins)"** مع إمكانية التنبيه اليدوي في حالات معينة.
- **📄 [MODIFY] `server/api.js`**: تعديل نقاط النهاية (Endpoints) لتقرأ وتكتب محلياً في SQLite، وتسجيل العمليات في جدول `sync_queue`.

## 📋 خطة التنفيذ خطوة بخطوة

### 🏁 المرحلة الأولى: تثبيت وتهيئة Electron
1. تثبيت الحزم المطلوبة لـ Electron و SQLite.
2. إنشاء ملفات `main.js` و `preload.js` الخاصة بـ Electron.
3. تعديل تكوين Vite ليدعم مسارات نسبية متوافقة مع Electron (`base: './'`).

### 💾 المرحلة الثانية: إعداد SQLite وقاعدة البيانات المحلية
1. كتابة سكربت بناء الجداول محلياً في SQLite.
2. مطابقة الأنواع البيانية بين PostgreSQL و SQLite.
3. توجيه خادم Express المحلي المدمج ليتصل بـ SQLite بدلاً من CockroachDB مباشرة عند العمل في وضع الأوفلاين أو دائماً (Offline-First).

### 🔄 المرحلة الثالثة: بناء نظام المزامنة (Synchronization Engine)
1. إنشاء جدول `sync_queue` محلياً لتسجيل العمليات:
   ```sql
   CREATE TABLE IF NOT EXISTS sync_queue (
     id INTEGER PRIMARY KEY AUTOINCREMENT,
     entity_name TEXT NOT NULL,
     record_id TEXT NOT NULL,
     operation TEXT NOT NULL, -- 'INSERT', 'UPDATE', 'DELETE'
     payload TEXT,            -- بيانات الحقل بصيغة JSON
     created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
     status TEXT DEFAULT 'pending' -- 'pending', 'synced', 'failed'
   );
   ```
2. تعديل الـ REST API في `server/api.js` ليقوم بحفظ العمليات محلياً وإضافتها للطابور.
3. إنشاء محرك المزامنة التلقائي في الخلفية ليقوم بالرفع والتحميل عند توفر الإنترنت.

### 🎨 المرحلة الرابعة: إضافة مؤشر حالة المزامنة والإنترنت في الواجهة
1. إضافة شريط علوي صغير أو أيقونة في القائمة الجانبية تُبين:
   - حالة الاتصال بالإنترنت (متصل 🟢 / غير متصل 🔴).
   - حالة المزامنة (تمت المزامنة بالكامل / يوجد 5 عمليات قيد الانتظار 🔄).
   - زر للمزامنة اليدوية الفورية.
